Getting started

Quickstart: create a new HTML5 file and copy over the top and bottom code from a preview template.

As you are building up your design it could be interesting to investigate individual HTML files from the preview and copy over the HTML blocks needed.
For example if you want portlets, look at the head tags and you will see that it uses a jQuery UI stylesheet. Then look at the bottom code to know which JavaScript files to use. The docs/js/ file always has the actual executing code whilst the extra folder holds the jQuery libraries. Not all pages use the same scripts.

Loading Bootstrap CSS

There are 10 minified CSS files and 10 non-minified files. It is best to use a minified version to decrease loading times.
You can opt to use the non-minified versions of the files and edit them directly if you don't want to recompile the code with LESS. Compiling with LESS is really easy and fun though! We'll get into this later.

These colors are available:
bright-green, flat-green, red, bordeaux, blue, purple-blue, aqua, hard-pink, mokka, gray 

<!-- Bootstrap core CSS -->
<link href="css/bootstrap.COLOR.min.css" rel="stylesheet">

Add the old IE support tags: 

<!-- HTML5 shim and Respond.js IE8 support of HTML5 elements and media queries -->
<!--[if lt IE 9]>
<script src="docs/js/html5shiv.js"></script>
<script src="docs/js/respond.min.js"></script>
<![endif]-->

Favicons (best create your own) and Font files: 

<!-- Favicons -->
<link rel="apple-touch-icon-precomposed" sizes="144x144" href="img/favicon_precomposed.jpg">
<link rel="icon" type="image/png" href="img/favicon.jpg">

<!-- Fonts -->
<link href='http://fonts.googleapis.com/css?family=Lato:100,300,400,700,100italic,300italic,400italic,700italic' rel='stylesheet' type='text/css'>

Now on the bottom of the page just above the body tag load jQuery and any JS files you need: the below method is used to make certain that when one CDN fails, the other will take over. 

<script src="https://code.jquery.com/jquery-1.10.2.min.js"></script>
<script>window.jQuery || document.write('<script src="http://ajax.googleapis.com/ajax/libs/jquery/1/jquery.min.js">\x3C/script>')</script>

Load the Bootstrap minified JS file: 

<script src="js/bootstrap.min.js"></script>

Optionally, load the jQuery UI JS file where needed BEFORE the Bootstrap JS file: 

<script src="extra/jquery-ui/jquery-ui.min.js"></script>
<script src="js/bootstrap.min.js"></script>

If you do this, be sure to load the jQuery UI CSS file BEFORE the bootstrap.css file: 

<link rel="stylesheet" href="extra/jquery-ui/jquery-ui.min.css">
<link href="css/bootstrap.min.css" rel="stylesheet" id="target">

Load any other scripts from extra, create any JS files you need for your project (using the docs/js as examples) - the code below is used on this manual page. 

<script src="extra/slim-scroll/jquery.slimscroll.min.js"></script>
<script src="js/portlets.js"></script>
<script src="js/slim-scroll.js"></script>

Overview of our LESS structure

How it works

We override the default Bootstrap styles in less/kvelle: existing styles will be rewritten here.
We create new Bootstrap styles in less/expansions/kvelle: new styles based on existing Bootstrap elements will be added here; we also create swatch-specific styles for plugins here. Each swatch has a subfolder: less/expansions/kvelle/dark, less/expansions/kvelle/darker, less/expansions/kvelle/light, less/expansions/kvelle/ligher, less/expansions/kvelle/neutral. Teh 12 extra color styles re found in less/expansions/kvelle/extra-colors.
All other folders in less/expansions relate to plugins and contain default plugin styling.

This means that there is a default style for each element and plugin which works without adding extra classes. Each element/plugin also has extra styles which are defined as described above. Everything is kept separate for maximum flexibility.

Jumbotron

Styles within jumbotrons are predefined, however you can override these with the main classes defined in core.less LINE 131.

The class decreased-padding reduces padding on the jumbotron so that you don't have the large padding where needed. The page titles use this class - the site title on the main index.html is an example of standard padding where this class is not used.

The class jumbotron-push provides extra margins to a jumbotron - index.html has an example of this (in combination with the site-title class).

Small text can be attained by using the small class.

The default jumbotron does not need any additional classes and has the default background color, lighter text and dark links. Theme color swatches are used by adding a class for example jumbotron-light to the existing jumbotron container.

mfp-move-from-top

docs/js/magnific-popup.js contains the scripts that call the CSS class when a visual effect is desired.

There are 2 ways to get a visual effect:
- add it manually in jQuery by adding for example mainClass: 'mfp-zoom-in-single' (class is found in CSS file)
- use data-effect="class-name"
Example LINE 136 (docs/js/magnific-popup.js): 

this.st.mainClass = this.st.el.attr('data-effect');

Zoom effects can also be obtained by using the built-in plugin settings:
Example LINE 74 (docs/js/magnific-popup.js): 

zoom: {
    enabled: true,
    duration: 300, // don't forget to change the duration also in CSS
    opener: function(element) {
        return element.find('img');
    }
}

Styling backgrounds

Several background colors are available: add it to the wrapper HTML tag containing the markup you want to give a colored background. Examples can be found in typography.html.
Page headers receive a style automatically depending on the text color that was given to the page header. The default page header color is the text color (a gray shade) but you can override it with the classes that apply text color.
These background color styles are defined in the type.less files found in less/expansions/kvelle and swatch subfolders.

For example, kvelle-background-main gives the background the main theme color.
When you add kvelle-color-lighter the page header will not have the default text color but it will have the color that corresponds with the lighter swatch of the theme color. You do not have to add this text color class to the same container as the background color class: color classes can be added to all HTML tags, allowing for great flexibility when applying text colors.
The page-header class will apply a bottom border color and bottom box shadow that fits this background color.

var body = $('body');
body.on('click', '.portlet > .panel > .panel-heading > .panel-options > .panel-collapse, .portlet > .panel > .panel-heading > .panel-options > .panel-expand', function (evt) {
//body.on('click', '.panel-collapse, .panel-expand', function (evt) { // if above doesn't work, no problems found yet
    evt.preventDefault();
    var el = $(this).closest(".portlet").find(".panel-body");
    //console.log(el);
    if ($(this).hasClass("panel-collapse")) {
        $(this).removeClass("panel-collapse").addClass("panel-expand");
        $(this).find('i').removeClass('fa-chevron-up').addClass('fa-chevron-down');
        el.slideUp(200);
    } else {
        $(this).removeClass("panel-expand").addClass("panel-collapse");
        $(this).find('i').removeClass('fa-chevron-down').addClass('fa-chevron-up');
        el.slideDown(200);
    }
});

body.on('click', '.portlet > .panel > .panel-heading > .panel-options > .panel-remove', function (evt) {
    evt.preventDefault();
    var el = $(this).closest(".portlet");
    el.fadeOut("normal", function() {
        $(this).remove();
    });
});
